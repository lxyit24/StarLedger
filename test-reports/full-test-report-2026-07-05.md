### 2.1 仪表盘 (Dashboard)

**功能描述**: 展示关键业务指标概览  
**测试结果**: ⚠️ 部分通过

| 测试项 | 状态 | 备注 |
|--------|:----:|------|
| 统计卡片显示 | ✅ | 服务器总数=1, 即将到期=0, 逾期账单=0, 本月总额=¥0.00 |
| 即将到期服务器表格 | ✅ | 显示"暂无数据" |
| 待处理账单表格 | ⚠️ | 数据显示但格式有问题 |

**发现的问题**:
1. 🔴 **日期格式原始**: 到期日显示 `2026-07-31T00:00:00Z` (ISO 原始格式)，应格式化为 `2026-07-31`
2. 🔴 **状态显示英文**: 账单状态显示 `paid` 而非中文 `已支付`
3. 🟡 **数据不一致**: 本月账单总额显示 ¥0.00，但待处理账单中有 ¥200 的账单

**优化方案**:
- 添加日期格式化函数，统一使用 `YYYY-MM-DD` 格式
- 添加状态枚举映射，将英文状态转为中文标签
- 核实统计逻辑，确保仪表盘数据与实际一致

### 2.4 合同管理模块

**功能描述**: 管理企业合同信息，包括合同创建、签署、到期跟踪  
**测试结果**: ❌ 不可用

| 测试项 | 状态 | 备注 |
|--------|:----:|------|
| 页面加载 | ✅ | 表格和筛选正常渲染 |
| 状态筛选下拉 | ✅ | 可展开 |
| 新增合同弹窗 | ✅ | 表单字段完整 |
| 创建合同 | ❌ | API 返回 403 Forbidden |
| 模块提示 | ⚠️ | 底部显示"该模块仅对特定租户类型可用" |

**发现的问题**:
1. 🔴 **API 403 错误**: `POST /api/v1/contracts` 返回 403，租户类型不匹配
2. 🔴 **体验矛盾**: 模块市场可以启用合同模块，但实际使用被租户类型限制，用户困惑
3. 🟡 **提示不明显**: "该模块仅对特定租户类型可用" 仅在页面底部，且创建操作无明确错误提示

**优化方案**:
- 方案A: 移除租户类型限制，所有类型租户均可使用
- 方案B: 在模块市场中明确标注适用的租户类型，不匹配时禁止启用
- 方案C: 403 错误时前端显示友好提示："当前租户类型不支持此功能，请联系管理员"
- 添加表单验证：必填字段未填时禁止提交并高亮提示

### 2.8 发票管理模块

**功能描述**: 发票开具、管理、查询  
**测试结果**: ❌ 创建失败

| 测试项 | 状态 | 备注 |
|--------|:----:|------|
| 统计卡片 | ✅ | 4个统计卡片正常显示 |
| 筛选器 | ✅ | 发票状态/类型筛选 |
| 新建发票弹窗 | ✅ | 表单字段完整，税额自动计算 |
| 创建发票 | ❌ | 外键约束错误 |
| 空数据提示 | ⚠️ | 显示英文 "No Data" |

**发现的问题**:
1. 🔴 **外键约束错误**: `FOREIGN KEY constraint failed (787)` - 发票 Schema 的 bill_id 外键约束导致创建失败
   - 原因: Invoice Schema 中 `bill_id` 字段可能设置了 `Optional()` 但数据库层面仍有 NOT NULL 约束
   - 或前端未传递必需的 bill_id 字段
2. 🔴 **中英文混杂**: "No Data" 应为 "暂无数据"
3. 🟡 **发票号码字段**: 标注为非必填但实际应自动生成

**优化方案**:
- 🔧 **紧急修复**: 检查 Invoice Schema 的 bill_id 字段，确保 `Optional()` 正确设置
- 重新生成 ent 代码: `go generate ./ent/...`
- 前端统一使用 Element Plus 的 `el-empty` 组件替代默认空数据提示
- 发票号码应由后端自动生成，前端不应要求用户填写

```
访问落地页 → 注册 → 登录 → 仪表盘 → 模块市场 → 各业务模块 → 系统管理
```

| 阶段 | 体验评分 | 关键问题 |
|------|:--------:|----------|
| 落地页浏览 | 8/10 | 内容丰富，视觉良好 |

3. **中英文混杂**: 严重影响专业感和国际化形象
